# Unnamed Messenger (пока не определился) — Консольный мессенджер на C++

Многопользовательский мессенджер с архитектурой клиент-сервер, написанный на **C++**. Сервер принимает подключения, управляет базой данных и рассылает уведомления о новых сообщениях всем подключённым клиентам в реальном времени.

---

## Текущие возможности

- **Регистрация и авторизация** пользователей
- **Создание личных чатов** с другими пользователями
- **Отправка и получение сообщений** в реальном времени
- **История сообщений** - загружается при входе в чат
- **Уведомления** о новых сообщениях из других чатов
- **Логирование** событий сервера в файл

---

## Архитектура

Проект разделён на три модуля:

| Модуль | Описание |
|---|---|
| `server` | TCP-сервер, обработчик пакетов, управление клиентами |
| `client` | Подключение к серверу, машина состояний UI, асинхронный слушатель |
| `common` | Общий сетевой протокол (`Packet`), логгер |

### Сетевой протокол

Клиент и сервер общаются по **JSON-пакетам** (`Packet`). Каждый пакет содержит тип (`PacketType`) и полезную нагрузку (`payload`).

**Типы пакетов:**

| Тип | Направление | Описание |
|---|---|---|
| `REGISTER` / `LOGIN` | Client → Server | Аутентификация |
| `CREATE_CHAT` | Client → Server | Создание нового диалога |
| `SEND_MESSAGE` | Client → Server | Отправка сообщения |
| `GET_CHATS` | Client → Server | Запрос списка чатов |
| `GET_CHAT_HISTORY` | Client → Server | Запрос истории чата |
| `CHAT_LIST_RESPONSE` | Server → Client | Список чатов пользователя |
| `HISTORY_RESPONSE` | Server → Client | История сообщений |
| `NEW_MESSAGE` | Server → Client | Push-уведомление о новом сообщении |
| `SUCCESS_RESPONSE` / `ERROR_RESPONSE` | Server → Client | Результат операции |

## Зависимости

| Зависимость | Версия | Назначение |
|---|---|---|
| **C++17** | ≥ 17 | Стандарт языка |
| **CMake** | ≥ 3.10 | Сборочная система |
| **SQLite3** | любая | База данных на стороне сервера |
| **nlohmann/json** | bundled | Сериализация пакетов |
| **POSIX Threads** | системная | Многопоточность |

### Установка зависимостей (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install cmake libsqlite3-dev build-essential
```

## Сборка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Walketit/Course3Work.git
cd Course3Work

# 2. Создать директорию для сборки
mkdir build && cd build

# 3. Сконфигурировать и собрать
cmake ..
cmake --build .

# 4. Запустить сервер (в отдельном терминале)
./messenger_server

# 5. Запустить клиент (в другом терминале)
./messenger_client
```

> **Примечание:** Сервер слушает порт **8080** на `127.0.0.1`. Файл базы данных `messenger.db` создаётся автоматически рядом с исполняемым файлом сервера.
